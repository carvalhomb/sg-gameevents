Game Events’s service documentation¶

REST API¶

GET /v1/sessions¶

Admin only. Requests a list of all active sessions.

Request Headers:
 
  • X-AUTH-TOKEN – A valid admin token obtained from GET /token
Status Codes:
  • 200 OK – Successful request
  • 400 Bad Request – Missing headers or parameters
  • 401 Unauthorized – Token not authorized to read all sessions
Response Headers:
 
  • Content-Type – application/json
  • X-Total-Count – Total number of results
Response JSON Array of Objects:
 
  • id (string) – Sessionid
  • created (string) – Timestamp of when the session was added to the local db
  • _links (string) – Links to the resource and related entities (if any)
GET /v1/version¶

Returns the current version of the API.

POST /v1/clients¶

Admin only. Adds a new client.

Request Headers:
 
  • X-AUTH-TOKEN – A valid admin token obtained from GET /token
Request JSON Object:
 
  • clientid (string) – human-readable id of the client to be created
  • api (string) – Plaintext version of the apikey
  • role (string) – Optional role (admin/normal). If not informed, a normal client will be created
Status Codes:
  • 201 Created – Client created successfully
  • 400 Bad Request – Not JSON object or missing parameters
  • 401 Unauthorized – Token not authorized
Response Headers:
 
  • Content-Type – application/json
  • X-Total-Count – the total number of results created
Response JSON Array of Objects:
 
  • id (string) – Internal id of the created client
  • clientid (string) – Human-readable id of the client
  • role (string) – Role of the created client
  • _links (string) – Link to the resource and related entities (if any)
POST /v1/token¶

Returns an authentication token to interact with the service. A non-admin client needs to provide a valid sessionid to get a token. An admin client can request a token without a valid “sessionid” and will be given access to all running sessions.

Request JSON Object:
 
  • clientid (string) – human-readable id of the client
  • apikey (string) – client’s apikey
  • sessionid (string) – UUID of the session. Optional for admins
Status Codes:
  • 200 OK – Successful request, returns the token
  • 400 Bad Request – Not JSON object or missing parameters
  • 401 Unauthorized – Wrong credentials
POST /v1/sessions/(sessionid)/events¶

Submits a new event related to the given sessionid. Client must be authorized to write to the session.

Parameters:
  • sessionid (string) –
Request Headers:
 
  • X-AUTH-TOKEN – A valid token obtained from GET /token
Request JSON Array of Objects:
 
  • events (string) – list of JSON game events
Status Codes:
  • 201 Created – Event created successfully
  • 400 Bad Request – Not JSON object or missing parameters
  • 401 Unauthorized – Token not authorized to write to session
  • 404 Not Found – Sessionid not found
Response Headers:
 
  • Content-Type – application/json
  • X-Total-Count – the total number of results created
Response JSON Array of Objects:
 
  • id (string) – Internal id of the created client
  • gameevent (string) – JSON representation of the event
  • _links (string) – Link to the resource and related entities (if any)
GET /v1/sessions/(sessionid)/events¶

Lists all game events associated to a gaming session. The user must be authorized to read the session.

Parameters:
  • sessionid (string) –
Request Headers:
 
  • X-AUTH-TOKEN – A valid token obtained from GET /token
Status Codes:
  • 200 OK – Successful request, returns the events
  • 401 Unauthorized – Token not authorized to read session
  • 404 Not Found – Sessionid not found
Response Headers:
 
  • Content-Type – application/json
  • X-Total-Count – the total number of results
Response JSON Array of Objects:
 
  • link (string) – Link to the resource
  • gameevent (string) – JSON representation of the event
